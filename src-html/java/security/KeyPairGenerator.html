038/**
039 * The KeyPairGenerator class is used to generate pairs of
040 * public and private keys. Key pair generators are constructed using the
041 * <code>getInstance</code> factory methods (static methods that
042 * return instances of a given class).
043 *
044 * <p>A Key pair generator for a particular algorithm creates a public/private
045 * key pair that can be used with this algorithm. It also associates
046 * algorithm-specific parameters with each of the generated keys.
047 *
048 * <p>There are two ways to generate a key pair: in an algorithm-independent
049 * manner, and in an algorithm-specific manner.
050 * The only difference between the two is the initialization of the object:
051 *
052 * <ul>
053 * <li><b>Algorithm-Independent Initialization</b>
054 * <p>All key pair generators share the concepts of a keysize and a
055 * source of randomness. The keysize is interpreted differently for different
056 * algorithms (e.g., in the case of the <i>DSA</i> algorithm, the keysize
057 * corresponds to the length of the modulus).
058 * There is an
059 * {@link #initialize(int, java.security.SecureRandom) initialize}
060 * method in this KeyPairGenerator class that takes these two universally
061 * shared types of arguments. There is also one that takes just a
062 * <code>keysize</code> argument, and uses the <code>SecureRandom</code>
063 * implementation of the highest-priority installed provider as the source
064 * of randomness. (If none of the installed providers supply an implementation
065 * of <code>SecureRandom</code>, a system-provided source of randomness is
066 * used.)
067 *
068 * <p>Since no other parameters are specified when you call the above
069 * algorithm-independent <code>initialize</code> methods, it is up to the
070 * provider what to do about the algorithm-specific parameters (if any) to be
071 * associated with each of the keys.
072 *
073 * <p>If the algorithm is the <i>DSA</i> algorithm, and the keysize (modulus
074 * size) is 512, 768, or 1024, then the <i>Sun</i> provider uses a set of
075 * precomputed values for the <code>p</code>, <code>q</code>, and
076 * <code>g</code> parameters. If the modulus size is not one of the above
077 * values, the <i>Sun</i> provider creates a new set of parameters. Other
078 * providers might have precomputed parameter sets for more than just the
079 * three modulus sizes mentioned above. Still others might not have a list of
080 * precomputed parameters at all and instead always create new parameter sets.
081 * <p>
082 *
083 * <li><b>Algorithm-Specific Initialization</b>
084 * <p>For situations where a set of algorithm-specific parameters already
085 * exists (e.g., so-called <i>community parameters</i> in DSA), there are two
086 * {@link #initialize(java.security.spec.AlgorithmParameterSpec)
087 * initialize} methods that have an <code>AlgorithmParameterSpec</code>
088 * argument. One also has a <code>SecureRandom</code> argument, while the
089 * the other uses the <code>SecureRandom</code>
090 * implementation of the highest-priority installed provider as the source
091 * of randomness. (If none of the installed providers supply an implementation
092 * of <code>SecureRandom</code>, a system-provided source of randomness is
093 * used.)
094 * </ul>
095 *
096 * <p>In case the client does not explicitly initialize the KeyPairGenerator
097 * (via a call to an <code>initialize</code> method), each provider must
098 * supply (and document) a default initialization.
099 * For example, the <i>Sun</i> provider uses a default modulus size (keysize)
100 * of 1024 bits.
101 *
102 * <p>Note that this class is abstract and extends from
103 * <code>KeyPairGeneratorSpi</code> for historical reasons.
104 * Application developers should only take notice of the methods defined in
105 * this <code>KeyPairGenerator</code> class; all the methods in
106 * the superclass are intended for cryptographic service providers who wish to
107 * supply their own implementations of key pair generators.
108 *
109 * <p> Every implementation of the Java platform is required to support the
110 * following standard <code>KeyPairGenerator</code> algorithms and keysizes in
111 * parentheses:
112 * <ul>
113 * <li><tt>DiffieHellman</tt> (1024)</li>
114 * <li><tt>DSA</tt> (1024)</li>
115 * <li><tt>RSA</tt> (1024, 2048)</li>
116 * </ul>
117 * These algorithms are described in the <a href=
118 * "{@docRoot}/../technotes/guides/security/StandardNames.html#KeyPairGenerator">
119 * KeyPairGenerator section</a> of the
120 * Java Cryptography Architecture Standard Algorithm Name Documentation.
121 * Consult the release documentation for your implementation to see if any
122 * other algorithms are supported.
123 *
124 * @author Benjamin Renaud
125 *
126 * @see java.security.spec.AlgorithmParameterSpec
127 */
128

167    private static KeyPairGenerator getInstance(Instance instance,
168            String algorithm) {
169        KeyPairGenerator kpg;
170        if (instance.impl instanceof KeyPairGenerator) {
171            kpg = (KeyPairGenerator)instance.impl;
172        } else {
173            KeyPairGeneratorSpi spi = (KeyPairGeneratorSpi)instance.impl;
174            kpg = new Delegate(spi, algorithm);
175        }
176        kpg.provider = instance.provider;
177
178        if (!skipDebug && pdebug != null) {
179            pdebug.println("KeyPairGenerator." + algorithm +
180                " algorithm from: " + kpg.provider.getName());
181        }
182
183        return kpg;
184    }

186    /**
187     * Returns a KeyPairGenerator object that generates public/private
188     * key pairs for the specified algorithm.
189     *
190     * <p> This method traverses the list of registered security Providers,
191     * starting with the most preferred Provider.
192     * A new KeyPairGenerator object encapsulating the
193     * KeyPairGeneratorSpi implementation from the first
194     * Provider that supports the specified algorithm is returned.
195     *
196     * <p> Note that the list of registered providers may be retrieved via
197     * the {@link Security#getProviders() Security.getProviders()} method.
198     *
199     * @param algorithm the standard string name of the algorithm.
200     * See the KeyPairGenerator section in the <a href=
201     * "{@docRoot}/../technotes/guides/security/StandardNames.html#KeyPairGenerator">
202     * Java Cryptography Architecture Standard Algorithm Name Documentation</a>
203     * for information about standard algorithm names.
204     *
205     * @return the new KeyPairGenerator object.
206     *
207     * @exception NoSuchAlgorithmException if no Provider supports a
208     *          KeyPairGeneratorSpi implementation for the
209     *          specified algorithm.
210     *
211     * @see Provider
212     */
213    public static KeyPairGenerator getInstance(String algorithm)
214            throws NoSuchAlgorithmException {
215        List<Service> list =
216                GetInstance.getServices("KeyPairGenerator", algorithm);
217        Iterator<Service> t = list.iterator();
218        if (t.hasNext() == false) {
219            throw new NoSuchAlgorithmException
220                (algorithm + " KeyPairGenerator not available");
221        }
222        // find a working Spi or KeyPairGenerator subclass
223        NoSuchAlgorithmException failure = null;
224        do {
225            Service s = t.next();
226            try {
227                Instance instance =
228                    GetInstance.getInstance(s, KeyPairGeneratorSpi.class);
229                if (instance.impl instanceof KeyPairGenerator) {
230                    return getInstance(instance, algorithm);
231                } else {
232                    return new Delegate(instance, t, algorithm);
233                }
234            } catch (NoSuchAlgorithmException e) {
235                if (failure == null) {
236                    failure = e;
237                }
238            }
239        } while (t.hasNext());
240        throw failure;
241    }

506    /*
507     * The following class allows providers to extend from KeyPairGeneratorSpi
508     * rather than from KeyPairGenerator. It represents a KeyPairGenerator
509     * with an encapsulated, provider-supplied SPI object (of type
510     * KeyPairGeneratorSpi).
511     * If the provider implementation is an instance of KeyPairGeneratorSpi,
512     * the getInstance() methods above return an instance of this class, with
513     * the SPI object encapsulated.
514     *
515     * Note: All SPI methods from the original KeyPairGenerator class have been
516     * moved up the hierarchy into a new class (KeyPairGeneratorSpi), which has
517     * been interposed in the hierarchy between the API (KeyPairGenerator)
518     * and its original parent (Object).
519     */
520
521    //
522    // error failover notes:
523    //
524    //  . we failover if the implementation throws an error during init
525    //    by retrying the init on other providers
526    //
527    //  . we also failover if the init succeeded but the subsequent call
528    //    to generateKeyPair() fails. In order for this to work, we need
529    //    to remember the parameters to the last successful call to init
530    //    and initialize() the next spi using them.
531    //
532    //  . although not specified, KeyPairGenerators could be thread safe,
533    //    so we make sure we do not interfere with that
534    //
535    //  . failover is not available, if:
536    //    . getInstance(algorithm, provider) was used
537    //    . a provider extends KeyPairGenerator rather than
538    //      KeyPairGeneratorSpi (JDK 1.1 style)
539    //    . once getProvider() is called
540    //
541
542    private static final class Delegate extends KeyPairGenerator {
543
544        // The provider implementation (delegate)
545        private volatile KeyPairGeneratorSpi spi;
546
547        private final Object lock = new Object();
548
549        private Iterator<Service> serviceIterator;
550
551        private final static int I_NONE   = 1;
552        private final static int I_SIZE   = 2;
553        private final static int I_PARAMS = 3;
554
555        private int initType;
556        private int initKeySize;
557        private AlgorithmParameterSpec initParams;
558        private SecureRandom initRandom;
559
560        // constructor
561        Delegate(KeyPairGeneratorSpi spi, String algorithm) {
562            super(algorithm);
563            this.spi = spi;
564        }
565
566        Delegate(Instance instance, Iterator<Service> serviceIterator,
567                String algorithm) {
568            super(algorithm);
569            spi = (KeyPairGeneratorSpi)instance.impl;
570            provider = instance.provider;
571            this.serviceIterator = serviceIterator;
572            initType = I_NONE;
573
574            if (!skipDebug && pdebug != null) {
575                pdebug.println("KeyPairGenerator." + algorithm +
576                    " algorithm from: " + provider.getName());
577            }
578        }
579
580        /**
581         * Update the active spi of this class and return the next
582         * implementation for failover. If no more implemenations are
583         * available, this method returns null. However, the active spi of
584         * this class is never set to null.
585         */
586        private KeyPairGeneratorSpi nextSpi(KeyPairGeneratorSpi oldSpi,
587                boolean reinit) {
588            synchronized (lock) {
589                // somebody else did a failover concurrently
590                // try that spi now
591                if ((oldSpi != null) && (oldSpi != spi)) {
592                    return spi;
593                }
594                if (serviceIterator == null) {
595                    return null;
596                }
597                while (serviceIterator.hasNext()) {
598                    Service s = serviceIterator.next();
599                    try {
600                        Object inst = s.newInstance(null);
601                        // ignore non-spis
602                        if (inst instanceof KeyPairGeneratorSpi == false) {
603                            continue;
604                        }
605                        if (inst instanceof KeyPairGenerator) {
606                            continue;
607                        }
608                        KeyPairGeneratorSpi spi = (KeyPairGeneratorSpi)inst;
609                        if (reinit) {
610                            if (initType == I_SIZE) {
611                                spi.initialize(initKeySize, initRandom);
612                            } else if (initType == I_PARAMS) {
613                                spi.initialize(initParams, initRandom);
614                            } else if (initType != I_NONE) {
615                                throw new AssertionError
616                                    ("KeyPairGenerator initType: " + initType);
617                            }
618                        }
619                        provider = s.getProvider();
620                        this.spi = spi;
621                        return spi;
622                    } catch (Exception e) {
623                        // ignore
624                    }
625                }
626                disableFailover();
627                return null;
628            }
629        }
630
631        void disableFailover() {
632            serviceIterator = null;
633            initType = 0;
634            initParams = null;
635            initRandom = null;
636        }
637
638        // engine method
639        public void initialize(int keysize, SecureRandom random) {
640            if (serviceIterator == null) {
641                spi.initialize(keysize, random);
642                return;
643            }
644            RuntimeException failure = null;
645            KeyPairGeneratorSpi mySpi = spi;
646            do {
647                try {
648                    mySpi.initialize(keysize, random);
649                    initType = I_SIZE;
650                    initKeySize = keysize;
651                    initParams = null;
652                    initRandom = random;
653                    return;
654                } catch (RuntimeException e) {
655                    if (failure == null) {
656                        failure = e;
657                    }
658                    mySpi = nextSpi(mySpi, false);
659                }
660            } while (mySpi != null);
661            throw failure;
662        }
663
664        // engine method
665        public void initialize(AlgorithmParameterSpec params,
666                SecureRandom random) throws InvalidAlgorithmParameterException {
667            if (serviceIterator == null) {
668                spi.initialize(params, random);
669                return;
670            }
671            Exception failure = null;
672            KeyPairGeneratorSpi mySpi = spi;
673            do {
674                try {
675                    mySpi.initialize(params, random);
676                    initType = I_PARAMS;
677                    initKeySize = 0;
678                    initParams = params;
679                    initRandom = random;
680                    return;
681                } catch (Exception e) {
682                    if (failure == null) {
683                        failure = e;
684                    }
685                    mySpi = nextSpi(mySpi, false);
686                }
687            } while (mySpi != null);
688            if (failure instanceof RuntimeException) {
689                throw (RuntimeException)failure;
690            }
691            // must be an InvalidAlgorithmParameterException
692            throw (InvalidAlgorithmParameterException)failure;
693        }
694
695        // engine method
696        public KeyPair generateKeyPair() {
697            if (serviceIterator == null) {
698                return spi.generateKeyPair();
699            }
700            RuntimeException failure = null;
701            KeyPairGeneratorSpi mySpi = spi;
702            do {
703                try {
704                    return mySpi.generateKeyPair();
705                } catch (RuntimeException e) {
706                    if (failure == null) {
707                        failure = e;
708                    }
709                    mySpi = nextSpi(mySpi, true);
710                }
711            } while (mySpi != null);
712            throw failure;
713        }
714    }
