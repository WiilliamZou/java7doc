039/**
040 * There is a single global LogManager object that is used to
041 * maintain a set of shared state about Loggers and log services.
042 * <p>
043 * This LogManager object:
044 * <ul>
045 * <li> Manages a hierarchical namespace of Logger objects.  All
046 *      named Loggers are stored in this namespace.
047 * <li> Manages a set of logging control properties.  These are
048 *      simple key-value pairs that can be used by Handlers and
049 *      other logging objects to configure themselves.
050 * </ul>
051 * <p>
052 * The global LogManager object can be retrieved using LogManager.getLogManager().
053 * The LogManager object is created during class initialization and
054 * cannot subsequently be changed.
055 * <p>
056 * At startup the LogManager class is located using the
057 * java.util.logging.manager system property.
058 * <p>
059 * By default, the LogManager reads its initial configuration from
060 * a properties file "lib/logging.properties" in the JRE directory.
061 * If you edit that property file you can change the default logging
062 * configuration for all uses of that JRE.
063 * <p>
064 * In addition, the LogManager uses two optional system properties that
065 * allow more control over reading the initial configuration:
066 * <ul>
067 * <li>"java.util.logging.config.class"
068 * <li>"java.util.logging.config.file"
069 * </ul>
070 * These two properties may be set via the Preferences API, or as
071 * command line property definitions to the "java" command, or as
072 * system property definitions passed to JNI_CreateJavaVM.
073 * <p>
074 * If the "java.util.logging.config.class" property is set, then the
075 * property value is treated as a class name.  The given class will be
076 * loaded, an object will be instantiated, and that object's constructor
077 * is responsible for reading in the initial configuration.  (That object
078 * may use other system properties to control its configuration.)  The
079 * alternate configuration class can use <tt>readConfiguration(InputStream)</tt>
080 * to define properties in the LogManager.
081 * <p>
082 * If "java.util.logging.config.class" property is <b>not</b> set,
083 * then the "java.util.logging.config.file" system property can be used
084 * to specify a properties file (in java.util.Properties format). The
085 * initial logging configuration will be read from this file.
086 * <p>
087 * If neither of these properties is defined then, as described
088 * above, the LogManager will read its initial configuration from
089 * a properties file "lib/logging.properties" in the JRE directory.
090 * <p>
091 * The properties for loggers and Handlers will have names starting
092 * with the dot-separated name for the handler or logger.
093 * <p>
094 * The global logging properties may include:
095 * <ul>
096 * <li>A property "handlers".  This defines a whitespace or comma separated
097 * list of class names for handler classes to load and register as
098 * handlers on the root Logger (the Logger named "").  Each class
099 * name must be for a Handler class which has a default constructor.
100 * Note that these Handlers may be created lazily, when they are
101 * first used.
102 *
103 * <li>A property "&lt;logger&gt;.handlers". This defines a whitespace or
104 * comma separated list of class names for handlers classes to
105 * load and register as handlers to the specified logger. Each class
106 * name must be for a Handler class which has a default constructor.
107 * Note that these Handlers may be created lazily, when they are
108 * first used.
109 *
110 * <li>A property "&lt;logger&gt;.useParentHandlers". This defines a boolean
111 * value. By default every logger calls its parent in addition to
112 * handling the logging message itself, this often result in messages
113 * being handled by the root logger as well. When setting this property
114 * to false a Handler needs to be configured for this logger otherwise
115 * no logging messages are delivered.
116 *
117 * <li>A property "config".  This property is intended to allow
118 * arbitrary configuration code to be run.  The property defines a
119 * whitespace or comma separated list of class names.  A new instance will be
120 * created for each named class.  The default constructor of each class
121 * may execute arbitrary code to update the logging configuration, such as
122 * setting logger levels, adding handlers, adding filters, etc.
123 * </ul>
124 * <p>
125 * Note that all classes loaded during LogManager configuration are
126 * first searched on the system class path before any user class path.
127 * That includes the LogManager class, any config classes, and any
128 * handler classes.
129 * <p>
130 * Loggers are organized into a naming hierarchy based on their
131 * dot separated names.  Thus "a.b.c" is a child of "a.b", but
132 * "a.b1" and a.b2" are peers.
133 * <p>
134 * All properties whose names end with ".level" are assumed to define
135 * log levels for Loggers.  Thus "foo.level" defines a log level for
136 * the logger called "foo" and (recursively) for any of its children
137 * in the naming hierarchy.  Log Levels are applied in the order they
138 * are defined in the properties file.  Thus level settings for child
139 * nodes in the tree should come after settings for their parents.
140 * The property name ".level" can be used to set the level for the
141 * root of the tree.
142 * <p>
143 * All methods on the LogManager object are multi-thread safe.
144 *
145 * @since 1.4
146*/
147

180    static {
181        AccessController.doPrivileged(new PrivilegedAction<Object>() {
182                public Object run() {
183                    String cname = null;
184                    try {
185                        cname = System.getProperty("java.util.logging.manager");
186                        if (cname != null) {
187                            try {
188                                Class clz = ClassLoader.getSystemClassLoader().loadClass(cname);
189                                manager = (LogManager) clz.newInstance();
190                            } catch (ClassNotFoundException ex) {
191                                Class clz = Thread.currentThread().getContextClassLoader().loadClass(cname);
192                                manager = (LogManager) clz.newInstance();
193                            }
194                        }
195                    } catch (Exception ex) {
196                        System.err.println("Could not load Logmanager \"" + cname + "\"");
197                        ex.printStackTrace();
198                    }
199                    if (manager == null) {
200                        manager = new LogManager();
201                    }
202
203                    // Create and retain Logger for the root of the namespace.
204                    manager.rootLogger = manager.new RootLogger();
205                    // since by design the global manager's userContext and
206                    // systemContext don't have their requiresDefaultLoggers
207                    // flag set - we make sure to add the root logger to
208                    // the global manager's default contexts here.
209                    manager.addLogger(manager.rootLogger);
210                    manager.systemContext.addLocalLogger(manager.rootLogger, false);
211                    manager.userContext.addLocalLogger(manager.rootLogger, false);
212
213                    // Adding the global Logger. Doing so in the Logger.<clinit>
214                    // would deadlock with the LogManager.<clinit>.
215                    Logger.global.setLogManager(manager);
216                    // Make sure the global logger will be registered in the
217                    // global manager's default contexts.
218                    manager.addLogger(Logger.global);
219                    manager.systemContext.addLocalLogger(Logger.global, false);
220                    manager.userContext.addLocalLogger(Logger.global, false);
221
222                    // We don't call readConfiguration() here, as we may be running
223                    // very early in the JVM startup sequence.  Instead readConfiguration
224                    // will be called lazily in getLogManager().
225                    return null;
226                }
227            });
228    }

231    // This private class is used as a shutdown hook.
232    // It does a "reset" to close all open handlers.
233    private class Cleaner extends Thread {
234
235        private Cleaner() {
236            /* Set context class loader to null in order to avoid
237             * keeping a strong reference to an application classloader.
238             */
239            this.setContextClassLoader(null);
240        }
241
242        public void run() {
243            // This is to ensure the LogManager.<clinit> is completed
244            // before synchronized block. Otherwise deadlocks are possible.
245            LogManager mgr = manager;
246
247            // If the global handlers haven't been initialized yet, we
248            // don't want to initialize them just so we can close them!
249            synchronized (LogManager.this) {
250                // Note that death is imminent.
251                deathImminent = true;
252                initializedGlobalHandlers = true;
253            }
254
255            // Do a reset to close all active handlers.
256            reset();
257        }
258    }

261    /**
262     * Protected constructor.  This is protected so that container applications
263     * (such as J2EE containers) can subclass the object.  It is non-public as
264     * it is intended that there only be one LogManager object, whose value is
265     * retrieved by calling Logmanager.getLogManager.

267    protected LogManager() {
268        this(checkSubclassPermissions());
269    }
270
271    private LogManager(Void checked) {
272
273        // Add a shutdown hook to close the global handlers.
274        try {
275            Runtime.getRuntime().addShutdownHook(new Cleaner());
276        } catch (IllegalStateException e) {
277            // If the VM is already shutting down,
278            // We do not need to register shutdownHook.
279        }
280    }

282    private static Void checkSubclassPermissions() {
283        final SecurityManager sm = System.getSecurityManager();
284        if (sm != null) {
285            // These permission will be checked in the LogManager constructor,
286            // in order to register the Cleaner() thread as a shutdown hook.
287            // Check them here to avoid the penalty of constructing the object
288            // etc...
289            sm.checkPermission(new RuntimePermission("shutdownHooks"));
290            sm.checkPermission(new RuntimePermission("setContextClassLoader"));
291        }
292        return null;
293    }

305    private void readPrimordialConfiguration() {
306        if (!readPrimordialConfiguration) {
307            synchronized (this) {
308                if (!readPrimordialConfiguration) {
309                    // If System.in/out/err are null, it's a good
310                    // indication that we're still in the
311                    // bootstrapping phase
312                    if (System.out == null) {
313                        return;
314                    }
315                    readPrimordialConfiguration = true;
316
317                    try {
318                        AccessController.doPrivileged(new PrivilegedExceptionAction<Void>() {
319                                public Void run() throws Exception {
320                                    readConfiguration();
321
322                                    // Platform loggers begin to delegate to java.util.logging.Logger
323                                    sun.util.logging.PlatformLogger.redirectPlatformLoggers();
324                                    return null;
325                                }
326                            });
327                    } catch (Exception ex) {
328                        // System.err.println("Can't read logging configuration:");
329                        // ex.printStackTrace();
330                    }
331                }
332            }
333        }
334    }

374    // LoggerContext maps from AppContext
375    private static WeakHashMap<Object, LoggerContext> contextsMap = null;

377    // Returns the LoggerContext for the user code (i.e. application or AppContext).
378    // Loggers are isolated from each AppContext.
379    private LoggerContext getUserContext() {
380        LoggerContext context = null;
381
382        SecurityManager sm = System.getSecurityManager();
383        JavaAWTAccess javaAwtAccess = SharedSecrets.getJavaAWTAccess();
384        if (sm != null && javaAwtAccess != null) {
385            // for each applet, it has its own LoggerContext isolated from others
386            final Object ecx = javaAwtAccess.getAppletContext();
387            if (ecx != null) {
388                synchronized (javaAwtAccess) {
389                    // find the AppContext of the applet code
390                    // will be null if we are in the main app context.
391                    if (contextsMap == null) {
392                        contextsMap = new WeakHashMap<>();
393                    }
394                    context = contextsMap.get(ecx);
395                    if (context == null) {
396                        // Create a new LoggerContext for the applet.
397                        // The new logger context has its requiresDefaultLoggers
398                        // flag set to true - so that these loggers will be
399                        // lazily added when the context is firt accessed.
400                        context = new LoggerContext(true);
401                        contextsMap.put(ecx, context);
402                    }
403                }
404            }
405        }
406        // for standalone app, return userContext
407        return context != null ? context : userContext;
408    }

410    private List<LoggerContext> contexts() {
411        List<LoggerContext> cxs = new ArrayList<>();
412        cxs.add(systemContext);
413        cxs.add(getUserContext());
414        return cxs;

417    // Find or create a specified logger instance. If a logger has
418    // already been created with the given name it is returned.
419    // Otherwise a new logger instance is created and registered
420    // in the LogManager global namespace.
421    // This method will always return a non-null Logger object.
422    // Synchronization is not required here. All synchronization for
423    // adding a new Logger object is handled by addLogger().
424    //
425    // This method must delegate to the LogManager implementation to
426    // add a new Logger or return the one that has been added previously
427    // as a LogManager subclass may override the addLogger, getLogger,
428    // readConfiguration, and other methods.
429    Logger demandLogger(String name, String resourceBundleName, Class<?> caller) {
430        Logger result = getLogger(name);
431        if (result == null) {
432            // only allocate the new logger once
433            Logger newLogger = new Logger(name, resourceBundleName, caller, false);
434            do {
435                if (addLogger(newLogger)) {
436                    // We successfully added the new Logger that we
437                    // created above so return it without refetching.
438                    return newLogger;
439                }
440
441                // We didn't add the new Logger that we created above
442                // because another thread added a Logger with the same
443                // name after our null check above and before our call
444                // to addLogger(). We have to refetch the Logger because
445                // addLogger() returns a boolean instead of the Logger
446                // reference itself. However, if the thread that created
447                // the other Logger is not holding a strong reference to
448                // the other Logger, then it is possible for the other
449                // Logger to be GC'ed after we saw it in addLogger() and
450                // before we can refetch it. If it has been GC'ed then
451                // we'll just loop around and try again.
452                result = getLogger(name);
453            } while (result == null);
454        }
455        return result;
456    }

458    Logger demandSystemLogger(String name, String resourceBundleName) {
459        // Add a system logger in the system context's namespace
460        final Logger sysLogger = systemContext.demandLogger(name, resourceBundleName);
461
462        // Add the system logger to the LogManager's namespace if not exist
463        // so that there is only one single logger of the given name.
464        // System loggers are visible to applications unless a logger of
465        // the same name has been added.
466        Logger logger;
467        do {
468            // First attempt to call addLogger instead of getLogger
469            // This would avoid potential bug in custom LogManager.getLogger
470            // implementation that adds a logger if does not exist
471            if (addLogger(sysLogger)) {
472                // successfully added the new system logger
473                logger = sysLogger;
474            } else {
475                logger = getLogger(name);
476            }
477        } while (logger == null);
478
479        // LogManager will set the sysLogger's handlers via LogManager.addLogger method.
480        if (logger != sysLogger && sysLogger.accessCheckedHandlers().length == 0) {
481            // if logger already exists but handlers not set
482            final Logger l = logger;
483            AccessController.doPrivileged(new PrivilegedAction<Void>() {
484                public Void run() {
485                    for (Handler hdl : l.accessCheckedHandlers()) {
486                        sysLogger.addHandler(hdl);
487                    }
488                    return null;
489                }
490            });
491        }
492        return sysLogger;
493    }

495    // LoggerContext maintains the logger namespace per context.
496    // The default LogManager implementation has one system context and user
497    // context.  The system context is used to maintain the namespace for
498    // all system loggers and is queried by the system code.  If a system logger
499    // doesn't exist in the user context, it'll also be added to the user context.
500    // The user context is queried by the user code and all other loggers are
501    // added in the user context.
502    static class LoggerContext {
503        // Table of named Loggers that maps names to Loggers.
504        private final Hashtable<String,LoggerWeakRef> namedLoggers = new Hashtable<>();
505        // Tree of named Loggers
506        private final LogNode root;
507        private final boolean requiresDefaultLoggers;
508        private LoggerContext() {
509            this(false);
510        }
511        private LoggerContext(boolean requiresDefaultLoggers) {
512            this.root = new LogNode(null, this);
513            this.requiresDefaultLoggers = requiresDefaultLoggers;
514        }
515
516        Logger demandLogger(String name, String resourceBundleName) {
517            // a LogManager subclass may have its own implementation to add and
518            // get a Logger.  So delegate to the LogManager to do the work.
519            return manager.demandLogger(name, resourceBundleName, null);
520        }
521
522
523        // Due to subtle deadlock issues getUserContext() no longer
524        // calls addLocalLogger(rootLogger);
525        // Therefore - we need to add the default loggers later on.
526        // Checks that the context is properly initialized
527        // This is necessary before calling e.g. find(name)
528        // or getLoggerNames()
529        //
530        private void ensureInitialized() {
531            if (requiresDefaultLoggers) {
532                // Ensure that the root and global loggers are set.
533                ensureDefaultLogger(manager.rootLogger);
534                ensureDefaultLogger(Logger.global);
535            }
536        }
537
538
539        synchronized Logger findLogger(String name) {
540            // ensure that this context is properly initialized before
541            // looking for loggers.
542            ensureInitialized();
543            LoggerWeakRef ref = namedLoggers.get(name);
544            if (ref == null) {
545                return null;
546            }
547            Logger logger = ref.get();
548            if (logger == null) {
549                // Hashtable holds stale weak reference
550                // to a logger which has been GC-ed.
551                ref.dispose();
552            }
553            return logger;
554        }
555
556        // This method is called before adding a logger to the
557        // context.
558        // 'logger' is the context that will be added.
559        // This method will ensure that the defaults loggers are added
560        // before adding 'logger'.
561        //
562        private void ensureAllDefaultLoggers(Logger logger) {
563            if (requiresDefaultLoggers) {
564                final String name = logger.getName();
565                if (!name.isEmpty()) {
566                    ensureDefaultLogger(manager.rootLogger);
567                }
568                if (!Logger.GLOBAL_LOGGER_NAME.equals(name)) {
569                    ensureDefaultLogger(Logger.global);
570                }
571            }
572        }
573
574        private void ensureDefaultLogger(Logger logger) {
575            // Used for lazy addition of root logger and global logger
576            // to a LoggerContext.
577
578            // This check is simple sanity: we do not want that this
579            // method be called for anything else than Logger.global
580            // or owner.rootLogger.
581            if (!requiresDefaultLoggers || logger == null
582                    || logger != Logger.global && logger != manager.rootLogger) {
583
584                // the case where we have a non null logger which is neither
585                // Logger.global nor manager.rootLogger indicates a serious
586                // issue - as ensureDefaultLogger should never be called
587                // with any other loggers than one of these two (or null - if
588                // e.g manager.rootLogger is not yet initialized)...
589                assert logger == null;
590
591                return;
592            }
593
594            // Adds the logger if it's not already there.
595            if (!namedLoggers.containsKey(logger.getName())) {
596                // It is important to prevent addLocalLogger to
597                // call ensureAllDefaultLoggers when we're in the process
598                // off adding one of those default loggers - as this would
599                // immediately cause a stack overflow.
600                // Therefore we must pass addDefaultLoggersIfNeeded=false,
601                // even if requiresDefaultLoggers is true.
602                addLocalLogger(logger, false);
603            }
604        }
605
606        boolean addLocalLogger(Logger logger) {
607            // no need to add default loggers if it's not required
608            return addLocalLogger(logger, requiresDefaultLoggers);
609        }
610
611        // Add a logger to this context.  This method will only set its level
612        // and process parent loggers.  It doesn't set its handlers.
613        synchronized boolean addLocalLogger(Logger logger, boolean addDefaultLoggersIfNeeded) {
614            // addDefaultLoggersIfNeeded serves to break recursion when adding
615            // default loggers. If we're adding one of the default loggers
616            // (we're being called from ensureDefaultLogger()) then
617            // addDefaultLoggersIfNeeded will be false: we don't want to
618            // call ensureAllDefaultLoggers again.
619            //
620            // Note: addDefaultLoggersIfNeeded can also be false when
621            //       requiresDefaultLoggers is false - since calling
622            //       ensureAllDefaultLoggers would have no effect in this case.
623            if (addDefaultLoggersIfNeeded) {
624                ensureAllDefaultLoggers(logger);
625            }
626
627            final String name = logger.getName();
628            if (name == null) {
629                throw new NullPointerException();
630            }
631
632            LoggerWeakRef ref = namedLoggers.get(name);
633            if (ref != null) {
634                if (ref.get() == null) {
635                    // It's possible that the Logger was GC'ed after a
636                    // drainLoggerRefQueueBounded() call so allow
637                    // a new one to be registered.
638                    ref.dispose();
639                } else {
640                    // We already have a registered logger with the given name.
641                    return false;
642                }
643            }
644
645            // We're adding a new logger.
646            // Note that we are creating a weak reference here.
647            ref = manager.new LoggerWeakRef(logger);
648            namedLoggers.put(name, ref);
649
650            // Apply any initial level defined for the new logger.
651            Level level = manager.getLevelProperty(name + ".level", null);
652            if (level != null) {
653                doSetLevel(logger, level);
654            }
655
656            processParentHandlers(logger, name);
657
658            // Find the new node and its parent.
659            LogNode node = getNode(name);
660            node.loggerRef = ref;
661            Logger parent = null;
662            LogNode nodep = node.parent;
663            while (nodep != null) {
664                LoggerWeakRef nodeRef = nodep.loggerRef;
665                if (nodeRef != null) {
666                    parent = nodeRef.get();
667                    if (parent != null) {
668                        break;
669                    }
670                }
671                nodep = nodep.parent;
672            }
673
674            if (parent != null) {
675                doSetParent(logger, parent);
676            }
677            // Walk over the children and tell them we are their new parent.
678            node.walkAndSetParent(logger);
679            // new LogNode is ready so tell the LoggerWeakRef about it
680            ref.setNode(node);
681            return true;
682        }
683
684        synchronized void removeLoggerRef(String name, LoggerWeakRef ref) {
685            if (namedLoggers.get(name) == ref) {
686                namedLoggers.remove(name);
687            }
688        }
689
690        synchronized Enumeration<String> getLoggerNames() {
691            // ensure that this context is properly initialized before
692            // returning logger names.
693            ensureInitialized();
694            return namedLoggers.keys();
695        }

697        // If logger.getUseParentHandlers() returns 'true' and any of the logger's
698        // parents have levels or handlers defined, make sure they are instantiated.
699        private void processParentHandlers(final Logger logger, final String name) {
700            AccessController.doPrivileged(new PrivilegedAction<Void>() {
701                public Void run() {
702                    if (logger != manager.rootLogger) {
703                        boolean useParent = manager.getBooleanProperty(name + ".useParentHandlers", true);
704                        if (!useParent) {
705                            logger.setUseParentHandlers(false);
706                        }
707                    }
708                    return null;
709                }
710            });
711
712            int ix = 1;
713            for (;;) {
714                int ix2 = name.indexOf(".", ix);
715                if (ix2 < 0) {
716                    break;
717                }
718                String pname = name.substring(0, ix2);
719                if (manager.getProperty(pname + ".level") != null ||
720                    manager.getProperty(pname + ".handlers") != null) {
721                    // This pname has a level/handlers definition.
722                    // Make sure it exists.
723                    demandLogger(pname, null);
724                }
725                ix = ix2+1;
726            }
727        }
728
729        // Gets a node in our tree of logger nodes.
730        // If necessary, create it.
731        LogNode getNode(String name) {
732            if (name == null || name.equals("")) {
733                return root;
734            }
735            LogNode node = root;
736            while (name.length() > 0) {
737                int ix = name.indexOf(".");
738                String head;
739                if (ix > 0) {
740                    head = name.substring(0, ix);
741                    name = name.substring(ix + 1);
742                } else {
743                    head = name;
744                    name = "";
745                }
746                if (node.children == null) {
747                    node.children = new HashMap<>();
748                }
749                LogNode child = node.children.get(head);
750                if (child == null) {
751                    child = new LogNode(node, this);
752                    node.children.put(head, child);
753                }
754                node = child;
755            }
756            return node;
757        }
758    }

760    static class SystemLoggerContext extends LoggerContext {
761        // Add a system logger in the system context's namespace as well as
762        // in the LogManager's namespace if not exist so that there is only
763        // one single logger of the given name.  System loggers are visible
764        // to applications unless a logger of the same name has been added.
765        Logger demandLogger(String name, String resourceBundleName) {
766            Logger result = findLogger(name);
767            if (result == null) {
768                // only allocate the new system logger once
769                Logger newLogger = new Logger(name, resourceBundleName, null, true);
770                do {
771                    if (addLocalLogger(newLogger)) {
772                        // We successfully added the new Logger that we
773                        // created above so return it without refetching.
774                        result = newLogger;
775                    } else {
776                        // We didn't add the new Logger that we created above
777                        // because another thread added a Logger with the same
778                        // name after our null check above and before our call
779                        // to addLogger(). We have to refetch the Logger because
780                        // addLogger() returns a boolean instead of the Logger
781                        // reference itself. However, if the thread that created
782                        // the other Logger is not holding a strong reference to
783                        // the other Logger, then it is possible for the other
784                        // Logger to be GC'ed after we saw it in addLogger() and
785                        // before we can refetch it. If it has been GC'ed then
786                        // we'll just loop around and try again.
787                        result = findLogger(name);
788                    }
789                } while (result == null);
790            }
791            return result;
792        }
793    }

841    // Package-level inner class.
842    // Helper class for managing WeakReferences to Logger objects.
843    //
844    // LogManager.namedLoggers
845    //     - has weak references to all named Loggers
846    //     - namedLoggers keeps the LoggerWeakRef objects for the named
847    //       Loggers around until we can deal with the book keeping for
848    //       the named Logger that is being GC'ed.
849    // LogManager.LogNode.loggerRef
850    //     - has a weak reference to a named Logger
851    //     - the LogNode will also keep the LoggerWeakRef objects for
852    //       the named Loggers around; currently LogNodes never go away.
853    // Logger.kids
854    //     - has a weak reference to each direct child Logger; this
855    //       includes anonymous and named Loggers
856    //     - anonymous Loggers are always children of the rootLogger
857    //       which is a strong reference; rootLogger.kids keeps the
858    //       LoggerWeakRef objects for the anonymous Loggers around
859    //       until we can deal with the book keeping.
860    //
861    final class LoggerWeakRef extends WeakReference<Logger> {
862        private String                name;       // for namedLoggers cleanup
863        private LogNode               node;       // for loggerRef cleanup
864        private WeakReference<Logger> parentRef;  // for kids cleanup
865        private boolean disposed = false;         // avoid calling dispose twice
866
867        LoggerWeakRef(Logger logger) {
868            super(logger, loggerRefQueue);
869
870            name = logger.getName();  // save for namedLoggers cleanup
871        }
872
873        // dispose of this LoggerWeakRef object
874        void dispose() {
875            // Avoid calling dispose twice. When a Logger is gc'ed, its
876            // LoggerWeakRef will be enqueued.
877            // However, a new logger of the same name may be added (or looked
878            // up) before the queue is drained. When that happens, dispose()
879            // will be called by addLocalLogger() or findLogger().
880            // Later when the queue is drained, dispose() will be called again
881            // for the same LoggerWeakRef. Marking LoggerWeakRef as disposed
882            // avoids processing the data twice (even though the code should
883            // now be reentrant).
884            synchronized(this) {
885                // Note to maintainers:
886                // Be careful not to call any method that tries to acquire
887                // another lock from within this block - as this would surely
888                // lead to deadlocks, given that dispose() can be called by
889                // multiple threads, and from within different synchronized
890                // methods/blocks.
891                if (disposed) return;
892                disposed = true;
893            }
894
895            final LogNode n = node;
896            if (n != null) {
897                // n.loggerRef can only be safely modified from within
898                // a lock on LoggerContext. removeLoggerRef is already
899                // synchronized on LoggerContext so calling
900                // n.context.removeLoggerRef from within this lock is safe.
901                synchronized (n.context) {
902                    // if we have a LogNode, then we were a named Logger
903                    // so clear namedLoggers weak ref to us
904                    n.context.removeLoggerRef(name, this);
905                    name = null;  // clear our ref to the Logger's name
906
907                    // LogNode may have been reused - so only clear
908                    // LogNode.loggerRef if LogNode.loggerRef == this
909                    if (n.loggerRef == this) {
910                        n.loggerRef = null;  // clear LogNode's weak ref to us
911                    }
912                    node = null;            // clear our ref to LogNode
913                }
914            }
915
916            if (parentRef != null) {
917                // this LoggerWeakRef has or had a parent Logger
918                Logger parent = parentRef.get();
919                if (parent != null) {
920                    // the parent Logger is still there so clear the
921                    // parent Logger's weak ref to us
922                    parent.removeChildLogger(this);
923                }
924                parentRef = null;  // clear our weak ref to the parent Logger
925            }
926        }
927
928        // set the node field to the specified value
929        void setNode(LogNode node) {
930            this.node = node;
931        }
932
933        // set the parentRef field to the specified value
934        void setParentRef(WeakReference<Logger> parentRef) {
935            this.parentRef = parentRef;
936        }
937    }

939    // Package-level method.
940    // Drain some Logger objects that have been GC'ed.
941    //
942    // drainLoggerRefQueueBounded() is called by addLogger() below
943    // and by Logger.getAnonymousLogger(String) so we'll drain up to
944    // MAX_ITERATIONS GC'ed Loggers for every Logger we add.
945    //
946    // On a WinXP VMware client, a MAX_ITERATIONS value of 400 gives
947    // us about a 50/50 mix in increased weak ref counts versus
948    // decreased weak ref counts in the AnonLoggerWeakRefLeak test.
949    // Here are stats for cleaning up sets of 400 anonymous Loggers:
950    //   - test duration 1 minute
951    //   - sample size of 125 sets of 400
952    //   - average: 1.99 ms
953    //   - minimum: 0.57 ms
954    //   - maximum: 25.3 ms
955    //
956    // The same config gives us a better decreased weak ref count
957    // than increased weak ref count in the LoggerWeakRefLeak test.
958    // Here are stats for cleaning up sets of 400 named Loggers:
959    //   - test duration 2 minutes
960    //   - sample size of 506 sets of 400
961    //   - average: 0.57 ms
962    //   - minimum: 0.02 ms
963    //   - maximum: 10.9 ms
964    //
965    private final static int MAX_ITERATIONS = 400;
966    final void drainLoggerRefQueueBounded() {
967        for (int i = 0; i < MAX_ITERATIONS; i++) {
968            if (loggerRefQueue == null) {
969                // haven't finished loading LogManager yet
970                break;
971            }
972
973            LoggerWeakRef ref = (LoggerWeakRef) loggerRefQueue.poll();
974            if (ref == null) {
975                break;
976            }
977            // a Logger object has been GC'ed so clean it up
978            ref.dispose();
979        }
980    }

985     * <p>

987     * newly created Logger.

1011            return false;

1091     * Reinitialize the logging properties and reread the logging configuration.
1092     * <p>
1093     * The same rules are used for locating the configuration properties
1094     * as are used at startup.  So normally the logging properties will
1095     * be re-read from the same file that was used at startup.
1096     * <P>
1097     * Any log level definitions in the new configuration file will be

1099     * <p>

1101     *
1102     * @exception  SecurityException  if a security manager exists and if
1103     *             the caller does not have LoggingPermission("control").
1104     * @exception  IOException if there are IO problems reading the configuration.
1105     */
1106    public void readConfiguration() throws IOException, SecurityException {
1107        checkPermission();
1108
1109        // if a configuration class is specified, load it and use it.
1110        String cname = System.getProperty("java.util.logging.config.class");
1111        if (cname != null) {
1112            try {
1113                // Instantiate the named class.  It is its constructor's
1114                // responsibility to initialize the logging configuration, by
1115                // calling readConfiguration(InputStream) with a suitable stream.
1116                try {
1117                    Class clz = ClassLoader.getSystemClassLoader().loadClass(cname);
1118                    clz.newInstance();
1119                    return;
1120                } catch (ClassNotFoundException ex) {
1121                    Class clz = Thread.currentThread().getContextClassLoader().loadClass(cname);
1122                    clz.newInstance();
1123                    return;
1124                }
1125            } catch (Exception ex) {
1126                System.err.println("Logging configuration class \"" + cname + "\" failed");
1127                System.err.println("" + ex);
1128                // keep going and useful config file.
1129            }
1130        }
1131
1132        String fname = System.getProperty("java.util.logging.config.file");
1133        if (fname == null) {
1134            fname = System.getProperty("java.home");
1135            if (fname == null) {
1136                throw new Error("Can't find java.home ??");
1137            }
1138            File f = new File(fname, "lib");
1139            f = new File(f, "logging.properties");
1140            fname = f.getCanonicalPath();
1141        }
1142        InputStream in = new FileInputStream(fname);
1143        BufferedInputStream bin = new BufferedInputStream(in);
1144        try {
1145            readConfiguration(bin);
1146        } finally {
1147            if (in != null) {
1148                in.close();
1149            }
1150        }
1151    }

1185    private void resetLogger(Logger logger) {

1188        for (int i = 0; i < targets.length; i++) {

1201        } else {

1206    // get a list of whitespace separated classnames from a property.
1207    private String[] parseClassNames(String propertyName) {
1208        String hands = getProperty(propertyName);
1209        if (hands == null) {
1210            return new String[0];
1211        }
1212        hands = hands.trim();
1213        int ix = 0;
1214        Vector<String> result = new Vector<>();
1215        while (ix < hands.length()) {
1216            int end = ix;
1217            while (end < hands.length()) {
1218                if (Character.isWhitespace(hands.charAt(end))) {
1219                    break;
1220                }
1221                if (hands.charAt(end) == ',') {
1222                    break;
1223                }
1224                end++;
1225            }
1226            String word = hands.substring(ix, end);
1227            ix = end+1;
1228            word = word.trim();
1229            if (word.length() == 0) {
1230                continue;
1231            }
1232            result.add(word);
1233        }
1234        return result.toArray(new String[result.size()]);
1235    }

1376        try {
1377            if (val != null) {
1378                Class clz = ClassLoader.getSystemClassLoader().loadClass(val);
1379                return (Formatter) clz.newInstance();
1380            }
1381        } catch (Exception ex) {
1382            // We got one of a variety of exceptions in creating the
1383            // class or creating an instance.
1384            // Drop through.
1385        }
1386        // We got an exception.  Return the defaultValue.
1387        return defaultValue;
1388    }

1390    // Private method to load the global handlers.
1391    // We do the real work lazily, when the global handlers
1392    // are first used.
1393    private synchronized void initializeGlobalHandlers() {
1394        if (initializedGlobalHandlers) {
1395            return;
1396        }
1397
1398        initializedGlobalHandlers = true;
1399
1400        if (deathImminent) {
1401            // Aaargh...
1402            // The VM is shutting down and our exit hook has been called.
1403            // Avoid allocating global handlers.
1404            return;
1405        }
1406        loadLoggerHandlers(rootLogger, null, "handlers");
1407    }

1431    // Nested class to represent a node in our tree of named loggers.
1432    private static class LogNode {
1433        HashMap<String,LogNode> children;
1434        LoggerWeakRef loggerRef;
1435        LogNode parent;
1436        final LoggerContext context;
1437
1438        LogNode(LogNode parent, LoggerContext context) {
1439            this.parent = parent;
1440            this.context = context;
1441        }
1442
1443        // Recursive method to walk the tree below a node and set
1444        // a new parent logger.
1445        void walkAndSetParent(Logger parent) {
1446            if (children == null) {
1447                return;
1448            }
1449            Iterator<LogNode> values = children.values().iterator();
1450            while (values.hasNext()) {
1451                LogNode node = values.next();
1452                LoggerWeakRef ref = node.loggerRef;
1453                Logger logger = (ref == null) ? null : ref.get();
1454                if (logger == null) {
1455                    node.walkAndSetParent(parent);
1456                } else {
1457                    doSetParent(logger, parent);
1458                }
1459            }
1460        }
1461    }

1463    // We use a subclass of Logger for the root logger, so
1464    // that we only instantiate the global handlers when they
1465    // are first needed.
1466    private final class RootLogger extends Logger {
1467        private RootLogger() {
1468            super("", null, null, true);
1469            setLevel(defaultLevel);
1470        }
1471
1472        @Override
1473        public void log(LogRecord record) {
1474            // Make sure that the global handlers have been instantiated.
1475            initializeGlobalHandlers();
1476            super.log(record);
1477        }
1478
1479        @Override
1480        public void addHandler(Handler h) {
1481            initializeGlobalHandlers();
1482            super.addHandler(h);
1483        }
1484
1485        @Override
1486        public void removeHandler(Handler h) {
1487            initializeGlobalHandlers();
1488            super.removeHandler(h);
1489        }
1490
1491        @Override
1492        Handler[] accessCheckedHandlers() {
1493            initializeGlobalHandlers();
1494            return super.accessCheckedHandlers();
1495        }
1496    }

1499    // Private method to be called when the configuration has
1500    // changed to apply any level settings to any pre-existing loggers.
1501    synchronized private void setLevelsOnExistingLoggers() {
1502        Enumeration<?> enum_ = props.propertyNames();
1503        while (enum_.hasMoreElements()) {
1504            String key = (String)enum_.nextElement();
1505            if (!key.endsWith(".level")) {
1506                // Not a level definition.
1507                continue;
1508            }
1509            int ix = key.length() - 6;
1510            String name = key.substring(0, ix);
1511            Level level = getLevelProperty(key, null);
1512            if (level == null) {
1513                System.err.println("Bad level value for property: " + key);
1514                continue;
1515            }
1516            for (LoggerContext cx : contexts()) {
1517                Logger l = cx.findLogger(name);
1518                if (l == null) {
1519                    continue;
1520                }
1521                l.setLevel(level);
1522            }
1523        }
1524    }
