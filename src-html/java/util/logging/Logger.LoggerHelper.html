041/**
042 * A Logger object is used to log messages for a specific
043 * system or application component.  Loggers are normally named,
044 * using a hierarchical dot-separated namespace.  Logger names
045 * can be arbitrary strings, but they should normally be based on
046 * the package name or class name of the logged component, such
047 * as java.net or javax.swing.  In addition it is possible to create
048 * "anonymous" Loggers that are not stored in the Logger namespace.
049 * <p>
050 * Logger objects may be obtained by calls on one of the getLogger
051 * factory methods.  These will either create a new Logger or
052 * return a suitable existing Logger. It is important to note that
053 * the Logger returned by one of the {@code getLogger} factory methods
054 * may be garbage collected at any time if a strong reference to the
055 * Logger is not kept.
056 * <p>
057 * Logging messages will be forwarded to registered Handler
058 * objects, which can forward the messages to a variety of
059 * destinations, including consoles, files, OS logs, etc.
060 * <p>
061 * Each Logger keeps track of a "parent" Logger, which is its
062 * nearest existing ancestor in the Logger namespace.
063 * <p>
064 * Each Logger has a "Level" associated with it.  This reflects
065 * a minimum Level that this logger cares about.  If a Logger's
066 * level is set to <tt>null</tt>, then its effective level is inherited
067 * from its parent, which may in turn obtain it recursively from its
068 * parent, and so on up the tree.
069 * <p>
070 * The log level can be configured based on the properties from the
071 * logging configuration file, as described in the description
072 * of the LogManager class.  However it may also be dynamically changed
073 * by calls on the Logger.setLevel method.  If a logger's level is
074 * changed the change may also affect child loggers, since any child
075 * logger that has <tt>null</tt> as its level will inherit its
076 * effective level from its parent.
077 * <p>
078 * On each logging call the Logger initially performs a cheap
079 * check of the request level (e.g., SEVERE or FINE) against the
080 * effective log level of the logger.  If the request level is
081 * lower than the log level, the logging call returns immediately.
082 * <p>
083 * After passing this initial (cheap) test, the Logger will allocate
084 * a LogRecord to describe the logging message.  It will then call a
085 * Filter (if present) to do a more detailed check on whether the
086 * record should be published.  If that passes it will then publish
087 * the LogRecord to its output Handlers.  By default, loggers also
088 * publish to their parent's Handlers, recursively up the tree.
089 * <p>
090 * Each Logger may have a ResourceBundle name associated with it.
091 * The named bundle will be used for localizing logging messages.
092 * If a Logger does not have its own ResourceBundle name, then
093 * it will inherit the ResourceBundle name from its parent,
094 * recursively up the tree.
095 * <p>
096 * Most of the logger output methods take a "msg" argument.  This
097 * msg argument may be either a raw value or a localization key.
098 * During formatting, if the logger has (or inherits) a localization
099 * ResourceBundle and if the ResourceBundle has a mapping for the msg
100 * string, then the msg string is replaced by the localized value.
101 * Otherwise the original msg string is used.  Typically, formatters use
102 * java.text.MessageFormat style formatting to format parameters, so
103 * for example a format string "{0} {1}" would format two parameters
104 * as strings.
105 * <p>
106 * When mapping ResourceBundle names to ResourceBundles, the Logger
107 * will first try to use the Thread's ContextClassLoader.  If that
108 * is null it will try the SystemClassLoader instead.  As a temporary
109 * transition feature in the initial implementation, if the Logger is
110 * unable to locate a ResourceBundle from the ContextClassLoader or
111 * SystemClassLoader the Logger will also search up the class stack
112 * and use successive calling ClassLoaders to try to locate a ResourceBundle.
113 * (This call stack search is to allow containers to transition to
114 * using ContextClassLoaders and is likely to be removed in future
115 * versions.)
116 * <p>
117 * Formatting (including localization) is the responsibility of
118 * the output Handler, which will typically call a Formatter.
119 * <p>
120 * Note that formatting need not occur synchronously.  It may be delayed
121 * until a LogRecord is actually written to an external sink.
122 * <p>
123 * The logging methods are grouped in five main categories:
124 * <ul>
125 * <li><p>
126 *     There are a set of "log" methods that take a log level, a message
127 *     string, and optionally some parameters to the message string.
128 * <li><p>
129 *     There are a set of "logp" methods (for "log precise") that are
130 *     like the "log" methods, but also take an explicit source class name
131 *     and method name.
132 * <li><p>
133 *     There are a set of "logrb" method (for "log with resource bundle")
134 *     that are like the "logp" method, but also take an explicit resource
135 *     bundle name for use in localizing the log message.
136 * <li><p>
137 *     There are convenience methods for tracing method entries (the
138 *     "entering" methods), method returns (the "exiting" methods) and
139 *     throwing exceptions (the "throwing" methods).
140 * <li><p>
141 *     Finally, there are a set of convenience methods for use in the
142 *     very simplest cases, when a developer simply wants to log a
143 *     simple string at a given log level.  These methods are named
144 *     after the standard Level names ("severe", "warning", "info", etc.)
145 *     and take a single argument, a message string.
146 * </ul>
147 * <p>
148 * For the methods that do not take an explicit source name and
149 * method name, the Logging framework will make a "best effort"
150 * to determine which class and method called into the logging method.
151 * However, it is important to realize that this automatically inferred
152 * information may only be approximate (or may even be quite wrong!).
153 * Virtual machines are allowed to do extensive optimizations when
154 * JITing and may entirely remove stack frames, making it impossible
155 * to reliably locate the calling class and method.
156 * <P>
157 * All methods on Logger are multi-thread safe.
158 * <p>
159 * <b>Subclassing Information:</b> Note that a LogManager class may
160 * provide its own implementation of named Loggers for any point in
161 * the namespace.  Therefore, any subclasses of Logger (unless they
162 * are implemented in conjunction with a new LogManager class) should
163 * take care to obtain a Logger instance from the LogManager class and
164 * should delegate operations such as "isLoggable" and "log(LogRecord)"
165 * to that instance.  Note that in order to intercept all logging
166 * output, subclasses need only override the log(LogRecord) method.
167 * All the other logging methods are implemented as calls on this
168 * log(LogRecord) method.
169 *
170 * @since 1.4
171 */

338    private static Logger demandLogger(String name, String resourceBundleName, Class<?> caller) {
339        LogManager manager = LogManager.getLogManager();
340        SecurityManager sm = System.getSecurityManager();
341        if (sm != null && !LoggerHelper.disableCallerCheck) {
342            if (caller.getClassLoader() == null) {
343                return manager.demandSystemLogger(name, resourceBundleName);
344            }
345        }
346        return manager.demandLogger(name, resourceBundleName, caller);
347        // ends up calling new Logger(name, resourceBundleName, caller)
348        // iff the logger doesn't exist already
349    }

352     * Find or create a logger for a named subsystem.  If a logger has
353     * already been created with the given name it is returned.  Otherwise
354     * a new logger is created.
355     * <p>
356     * If a new logger is created its log level will be configured
357     * based on the LogManager configuration and it will configured
358     * to also send logging output to its parent's Handlers.  It will
359     * be registered in the LogManager global namespace.
360     * <p>
361     * Note: The LogManager may only retain a weak reference to the newly
362     * created Logger. It is important to understand that a previously
363     * created Logger with the given name may be garbage collected at any
364     * time if there is no strong reference to the Logger. In particular,
365     * this means that two back-to-back calls like
366     * {@code getLogger("MyLogger").log(...)} may use different Logger
367     * objects named "MyLogger" if there is no strong reference to the
368     * Logger named "MyLogger" elsewhere in the program.
369     *
370     * @param   name            A name for the logger.  This should
371     *                          be a dot-separated name and should normally
372     *                          be based on the package name or class name
373     *                          of the subsystem, such as java.net
374     *                          or javax.swing
375     * @return a suitable Logger
376     * @throws NullPointerException if the name is null.
377     */

379    // Synchronization is not required here. All synchronization for
380    // adding a new Logger object is handled by LogManager.addLogger().
381    @CallerSensitive
382    public static Logger getLogger(String name) {
383        // This method is intentionally not a wrapper around a call
384        // to getLogger(name, resourceBundleName). If it were then
385        // this sequence:
386        //
387        //     getLogger("Foo", "resourceBundleForFoo");
388        //     getLogger("Foo");
389        //
390        // would throw an IllegalArgumentException in the second call
391        // because the wrapper would result in an attempt to replace
392        // the existing "resourceBundleForFoo" with null.
393        return demandLogger(name, null, Reflection.getCallerClass());
394    }

396    /**
397     * Find or create a logger for a named subsystem.  If a logger has
398     * already been created with the given name it is returned.  Otherwise
399     * a new logger is created.
400     * <p>
401     * If a new logger is created its log level will be configured
402     * based on the LogManager and it will configured to also send logging
403     * output to its parent's Handlers.  It will be registered in
404     * the LogManager global namespace.
405     * <p>
406     * Note: The LogManager may only retain a weak reference to the newly
407     * created Logger. It is important to understand that a previously
408     * created Logger with the given name may be garbage collected at any
409     * time if there is no strong reference to the Logger. In particular,
410     * this means that two back-to-back calls like
411     * {@code getLogger("MyLogger", ...).log(...)} may use different Logger
412     * objects named "MyLogger" if there is no strong reference to the
413     * Logger named "MyLogger" elsewhere in the program.
414     * <p>
415     * If the named Logger already exists and does not yet have a
416     * localization resource bundle then the given resource bundle
417     * name is used.  If the named Logger already exists and has
418     * a different resource bundle name then an IllegalArgumentException
419     * is thrown.
420     * <p>
421     * @param   name    A name for the logger.  This should
422     *                          be a dot-separated name and should normally
423     *                          be based on the package name or class name
424     *                          of the subsystem, such as java.net
425     *                          or javax.swing
426     * @param   resourceBundleName  name of ResourceBundle to be used for localizing
427     *                          messages for this logger. May be <CODE>null</CODE> if none of
428     *                          the messages require localization.
429     * @return a suitable Logger
430     * @throws MissingResourceException if the resourceBundleName is non-null and
431     *             no corresponding resource can be found.
432     * @throws IllegalArgumentException if the Logger already exists and uses
433     *             a different resource bundle name.
434     * @throws NullPointerException if the name is null.
435     */
436
437    // Synchronization is not required here. All synchronization for
438    // adding a new Logger object is handled by LogManager.addLogger().
439    @CallerSensitive
440    public static Logger getLogger(String name, String resourceBundleName) {
441        Class<?> callerClass = Reflection.getCallerClass();
442        Logger result = demandLogger(name, resourceBundleName, callerClass);
443
444        if (result.resourceBundleName == null) {
445            // We haven't set a bundle name yet on the Logger, so it's ok to proceed.
446
447            // We have to set the callers ClassLoader here in case demandLogger
448            // above found a previously created Logger.  This can happen, for
449            // example, if Logger.getLogger(name) is called and subsequently
450            // Logger.getLogger(name, resourceBundleName) is called.  In this case
451            // we won't necessarily have the correct classloader saved away, so
452            // we need to set it here, too.
453
454            // Note: we may get a MissingResourceException here.
455            result.setupResourceInfo(resourceBundleName, callerClass);
456        } else if (!result.resourceBundleName.equals(resourceBundleName)) {
457            // We already had a bundle name on the Logger and we're trying
458            // to change it here which is not allowed.
459            throw new IllegalArgumentException(result.resourceBundleName +
460                                " != " + resourceBundleName);
461        }
462        return result;
463    }

512     * <p>
513     * Even although the new logger is anonymous, it is configured
514     * to have the root logger ("") as its parent.  This means that
515     * by default it inherits its effective level and handlers
516     * from the root logger.
517     * <p>
518     * @param   resourceBundleName  name of ResourceBundle to be used for localizing
519     *                          messages for this logger.
520     *          May be null if none of the messages require localization.
521     * @return a newly created private Logger

527    // adding a new anonymous Logger object is handled by doSetParent().
528    @CallerSensitive
529    public static Logger getAnonymousLogger(String resourceBundleName) {
530        LogManager manager = LogManager.getLogManager();
531        // cleanup some Loggers that have been GC'ed
532        manager.drainLoggerRefQueueBounded();
533        Logger result = new Logger(null, resourceBundleName,
534                                   Reflection.getCallerClass(), false);
535        result.anonymous = true;
536        Logger root = manager.getLogger("");
537        result.doSetParent(root);
538        return result;
539    }

589    /**
590     * Log a LogRecord.
591     * <p>
592     * All the other logging methods in this class call through
593     * this method to actually perform any logging.  Subclasses can
594     * override this single method to capture all log activity.
595     *
596     * @param record the LogRecord to be published
597     */
598    public void log(LogRecord record) {
599        if (record.getLevel().intValue() < levelValue || levelValue == offValue) {
600            return;
601        }
602        Filter theFilter = filter;
603        if (theFilter != null && !theFilter.isLoggable(record)) {
604            return;
605        }
606
607        // Post the LogRecord to all our Handlers, and then to
608        // our parents' handlers, all the way up the tree.
609
610        Logger logger = this;
611        while (logger != null) {
612            final Handler[] loggerHandlers = isSystemLogger
613                ? logger.accessCheckedHandlers()
614                : logger.getHandlers();
615            for (Handler handler : loggerHandlers) {
616                handler.publish(record);
617            }
618
619            final boolean useParentHdls = isSystemLogger
620                ? logger.useParentHandlers
621                : logger.getUseParentHandlers();
622
623            if (!useParentHdls) {
624                break;
625            }
626
627            logger = isSystemLogger ? logger.parent : logger.getParent();
628        }
629    }

631    // private support method for logging.
632    // We fill in the logger name, resource bundle name, and
633    // resource bundle and then call "void log(LogRecord)".
634    private void doLog(LogRecord lr) {
635        lr.setLoggerName(name);
636        String ebname = getEffectiveResourceBundleName();
637        if (ebname != null && !ebname.equals(SYSTEM_LOGGER_RB_NAME)) {
638            lr.setResourceBundleName(ebname);
639            lr.setResourceBundle(findResourceBundle(ebname, true));
640        }
641        log(lr);
642    }

649    /**
650     * Log a message, with no arguments.
651     * <p>
652     * If the logger is currently enabled for the given message
653     * level then the given message is forwarded to all the
654     * registered output Handler objects.
655     * <p>
656     * @param   level   One of the message level identifiers, e.g., SEVERE

1402    /**
1403     * Private utility method to map a resource bundle name to an
1404     * actual resource bundle, using a simple one-entry cache.
1405     * Returns null for a null name.
1406     * May also return null if we can't find the resource bundle and
1407     * there is no suitable previous cached value.
1408     *
1409     * @param name the ResourceBundle to locate
1410     * @param userCallersClassLoader if true search using the caller's ClassLoader
1411     * @return ResourceBundle specified by name or null if not found
1412     */
1413    private synchronized ResourceBundle findResourceBundle(String name,
1414                                                           boolean useCallersClassLoader) {
1415        // For all lookups, we first check the thread context class loader
1416        // if it is set.  If not, we use the system classloader.  If we
1417        // still haven't found it we use the callersClassLoaderRef if it
1418        // is set and useCallersClassLoader is true.  We set
1419        // callersClassLoaderRef initially upon creating the logger with a
1420        // non-null resource bundle name.
1421
1422        // Return a null bundle for a null name.
1423        if (name == null) {
1424            return null;
1425        }
1426
1427        Locale currentLocale = Locale.getDefault();
1428
1429        // Normally we should hit on our simple one entry cache.
1430        if (catalog != null && currentLocale.equals(catalogLocale)
1431                && name.equals(catalogName)) {
1432            return catalog;
1433        }
1434
1435        if (name.equals(SYSTEM_LOGGER_RB_NAME)) {
1436            catalog = findSystemResourceBundle(currentLocale);
1437            catalogName = name;
1438            catalogLocale = currentLocale;
1439            return catalog;
1440        }
1441
1442        // Use the thread's context ClassLoader.  If there isn't one, use the
1443        // {@linkplain java.lang.ClassLoader#getSystemClassLoader() system ClassLoader}.
1444        ClassLoader cl = Thread.currentThread().getContextClassLoader();
1445        if (cl == null) {
1446            cl = ClassLoader.getSystemClassLoader();
1447        }
1448        try {
1449            catalog = ResourceBundle.getBundle(name, currentLocale, cl);
1450            catalogName = name;
1451            catalogLocale = currentLocale;
1452            return catalog;
1453        } catch (MissingResourceException ex) {
1454            // We can't find the ResourceBundle in the default
1455            // ClassLoader.  Drop through.
1456        }
1457
1458        if (useCallersClassLoader) {
1459            // Try with the caller's ClassLoader
1460            ClassLoader callersClassLoader = getCallersClassLoader();
1461            if (callersClassLoader != null && callersClassLoader != cl) {
1462                try {
1463                    catalog = ResourceBundle.getBundle(name, currentLocale,
1464                                                       callersClassLoader);
1465                    catalogName = name;
1466                    catalogLocale = currentLocale;
1467                    return catalog;
1468                } catch (MissingResourceException ex) {
1469                }
1470            }
1471        }
1472
1473        // If -Djdk.logging.allowStackWalkSearch=true is set,
1474        // does stack walk to search for the resource bundle
1475        if (LoggerHelper.allowStackWalkSearch) {
1476            return findResourceBundleFromStack(name, currentLocale, cl);
1477        } else {
1478            return null;
1479        }
1480    }

1482    /**
1483     * This method will fail when running with a VM that enforces caller-sensitive
1484     * methods and only allows to get the immediate caller.
1485     */
1486    @CallerSensitive
1487    private synchronized ResourceBundle findResourceBundleFromStack(String name,
1488                                                                    Locale locale,
1489                                                                    ClassLoader cl)
1490    {
1491        for (int ix = 0; ; ix++) {
1492            Class<?> clz = sun.reflect.Reflection.getCallerClass(ix);
1493            if (clz == null) {
1494                break;
1495            }
1496            ClassLoader cl2 = clz.getClassLoader();
1497            if (cl2 == null) {
1498                cl2 = ClassLoader.getSystemClassLoader();
1499            }
1500            if (cl == cl2) {
1501                // We've already checked this classloader.
1502                continue;
1503            }
1504            cl = cl2;
1505            try {
1506                catalog = ResourceBundle.getBundle(name, locale, cl);
1507                catalogName = name;
1508                catalogLocale = locale;
1509                return catalog;
1510            } catch (MissingResourceException ex) {
1511            }
1512        }
1513        return null;
1514    }

1516    // Private utility method to initialize our one entry
1517    // resource bundle name cache and the callers ClassLoader
1518    // Note: for consistency reasons, we are careful to check
1519    // that a suitable ResourceBundle exists before setting the
1520    // resourceBundleName field.
1521    // Synchronized to prevent races in setting the fields.
1522    private synchronized void setupResourceInfo(String name,
1523                                                Class<?> callersClass) {
1524        if (name == null) {
1525            return;
1526        }
1527
1528        setCallersClassLoaderRef(callersClass);
1529        if (isSystemLogger && getCallersClassLoader() != null) {
1530            checkPermission();
1531        }
1532        if (findResourceBundle(name, true) == null) {
1533            // We've failed to find an expected ResourceBundle.
1534            // unset the caller's ClassLoader since we were unable to find the
1535            // the bundle using it
1536            this.callersClassLoaderRef = null;
1537            throw new MissingResourceException("Can't find " + name + " bundle",
1538                                                name, "");
1539        }
1540        resourceBundleName = name;
1541    }

1544     * Return the parent for this Logger.
1545     * <p>
1546     * This method returns the nearest extant parent in the namespace.
1547     * Thus if a Logger is called "a.b.c.d", and a Logger called "a.b"
1548     * has been created but no logger "a.b.c" exists, then a call of
1549     * getParent on the Logger "a.b.c.d" will return the Logger "a.b".
1550     * <p>
1551     * The result will be null if it is called on the root Logger
1552     * in the namespace.
1553     *
1554     * @return nearest existing parent Logger
1555     */
1556    public Logger getParent() {
1557        // Note: this used to be synchronized on treeLock.  However, this only
1558        // provided memory semantics, as there was no guarantee that the caller
1559        // would synchronize on treeLock (in fact, there is no way for external
1560        // callers to so synchronize).  Therefore, we have made parent volatile
1561        // instead.
1562        return parent;
1563    }

1566     * Set the parent for this Logger.  This method is used by
1567     * the LogManager to update a Logger when the namespace changes.
1568     * <p>
1569     * It should not be called from application code.
1570     * <p>
1571     * @param  parent   the new parent logger
1572     * @exception  SecurityException  if a security manager exists and if
1573     *             the caller does not have LoggingPermission("control").
1574     */
1575    public void setParent(Logger parent) {
1576        if (parent == null) {
1577            throw new NullPointerException();
1578        }
1579        if (manager == null) {
1580            manager = LogManager.getLogManager();
1581        }
1582        manager.checkPermission();
1583        doSetParent(parent);

1586    // Private method to do the work for parenting a child
1587    // Logger onto a parent logger.
1588    private void doSetParent(Logger newParent) {
1589
1590        // System.err.println("doSetParent \"" + getName() + "\" \""
1591        //                              + newParent.getName() + "\"");
1592
1593        synchronized (treeLock) {
1594
1595            // Remove ourself from any previous parent.
1596            LogManager.LoggerWeakRef ref = null;
1597            if (parent != null) {
1598                // assert parent.kids != null;
1599                for (Iterator<LogManager.LoggerWeakRef> iter = parent.kids.iterator(); iter.hasNext(); ) {
1600                    ref = iter.next();
1601                    Logger kid =  ref.get();
1602                    if (kid == this) {
1603                        // ref is used down below to complete the reparenting
1604                        iter.remove();
1605                        break;
1606                    } else {
1607                        ref = null;
1608                    }
1609                }
1610                // We have now removed ourself from our parents' kids.
1611            }
1612
1613            // Set our new parent.
1614            parent = newParent;
1615            if (parent.kids == null) {
1616                parent.kids = new ArrayList<>(2);
1617            }
1618            if (ref == null) {
1619                // we didn't have a previous parent
1620                ref = manager.new LoggerWeakRef(this);
1621            }
1622            ref.setParentRef(new WeakReference<Logger>(parent));
1623            parent.kids.add(ref);
1624
1625            // As a result of the reparenting, the effective level
1626            // may have changed for us and our children.
1627            updateEffectiveLevel();
1628
1629        }
1630    }

1632    // Package-level method.
1633    // Remove the weak reference for the specified child Logger from the
1634    // kid list. We should only be called from LoggerWeakRef.dispose().
1635    final void removeChildLogger(LogManager.LoggerWeakRef child) {
1636        synchronized (treeLock) {
1637            for (Iterator<LogManager.LoggerWeakRef> iter = kids.iterator(); iter.hasNext(); ) {
1638                LogManager.LoggerWeakRef ref = iter.next();
1639                if (ref == child) {
1640                    iter.remove();
1641                    return;
1642                }
1643            }
1644        }
1645    }

1647    // Recalculate the effective level for this node and
1648    // recursively for our children.
1649
1650    private void updateEffectiveLevel() {
1651        // assert Thread.holdsLock(treeLock);
1652
1653        // Figure out our current effective level.
1654        int newLevelValue;
1655        if (levelObject != null) {
1656            newLevelValue = levelObject.intValue();
1657        } else {
1658            if (parent != null) {
1659                newLevelValue = parent.levelValue;
1660            } else {
1661                // This may happen during initialization.
1662                newLevelValue = Level.INFO.intValue();
1663            }
1664        }
1665
1666        // If our effective value hasn't changed, we're done.
1667        if (levelValue == newLevelValue) {
1668            return;
1669        }
1670
1671        levelValue = newLevelValue;
1672
1673        // System.err.println("effective level: \"" + getName() + "\" := " + level);
1674
1675        // Recursively update the level on each of our kids.
1676        if (kids != null) {
1677            for (int i = 0; i < kids.size(); i++) {
1678                LogManager.LoggerWeakRef ref = kids.get(i);
1679                Logger kid =  ref.get();
1680                if (kid != null) {
1681                    kid.updateEffectiveLevel();
1682                }
1683            }
1684        }
1685    }

1688    // Private method to get the potentially inherited
1689    // resource bundle name for this Logger.
1690    // May return null
1691    private String getEffectiveResourceBundleName() {
1692        Logger target = this;
1693        while (target != null) {
1694            final String rbn = isSystemLogger
1695                // ancestor of a system logger is expected to be a system logger.
1696                // ignore resource bundle name if it's not.
1697                ? (target.isSystemLogger ? target.resourceBundleName : null)
1698                : target.getResourceBundleName();
1699            if (rbn != null) {
1700                return rbn;
1701            }
1702            target = isSystemLogger ? target.parent : target.getParent();
1703        }
1704        return null;
1705    }
