029/**
030 * Comprehensive information about the database as a whole.
031 * <P>
032 * This interface is implemented by driver vendors to let users know the capabilities
033 * of a Database Management System (DBMS) in combination with
034 * the driver based on JDBC<sup><font size=-2>TM</font></sup> technology
035 * ("JDBC driver") that is used with it.  Different relational DBMSs often support
036 * different features, implement features in different ways, and use different
037 * data types.  In addition, a driver may implement a feature on top of what the
038 * DBMS offers.  Information returned by methods in this interface applies
039 * to the capabilities of a particular driver and a particular DBMS working
040 * together. Note that as used in this documentation, the term "database" is
041 * used generically to refer to both the driver and DBMS.
042 * <P>
043 * A user for this interface is commonly a tool that needs to discover how to
044 * deal with the underlying DBMS.  This is especially true for applications
045 * that are intended to be used with more than one DBMS. For example, a tool might use the method
046 * <code>getTypeInfo</code> to find out what data types can be used in a
047 * <code>CREATE TABLE</code> statement.  Or a user might call the method
048 * <code>supportsCorrelatedSubqueries</code> to see if it is possible to use
049 * a correlated subquery or <code>supportsBatchUpdates</code> to see if it is
050 * possible to use batch updates.
051 * <P>
052 * Some <code>DatabaseMetaData</code> methods return lists of information
053 * in the form of <code>ResultSet</code> objects.
054 * Regular <code>ResultSet</code> methods, such as
055 * <code>getString</code> and <code>getInt</code>, can be used
056 * to retrieve the data from these <code>ResultSet</code> objects.  If
057 * a given form of metadata is not available, an empty <code>ResultSet</code>
058 * will be returned. Additional columns beyond the columns defined to be
059 * returned by the <code>ResultSet</code> object for a given method
060 * can be defined by the JDBC driver vendor and must be accessed
061 * by their <B>column label</B>.
062 * <P>
063 * Some <code>DatabaseMetaData</code> methods take arguments that are
064 * String patterns.  These arguments all have names such as fooPattern.
065 * Within a pattern String, "%" means match any substring of 0 or more
066 * characters, and "_" means match any one character. Only metadata
067 * entries matching the search pattern are returned. If a search pattern
068 * argument is set to <code>null</code>, that argument's criterion will
069 * be dropped from the search.
070 * <P>
071 */
