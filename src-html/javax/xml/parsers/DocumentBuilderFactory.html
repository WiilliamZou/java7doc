062    /**
063     * Obtain a new instance of a
064     * <code>DocumentBuilderFactory</code>. This static method creates
065     * a new factory instance.
066     * This method uses the following ordered lookup procedure to determine
067     * the <code>DocumentBuilderFactory</code> implementation class to
068     * load:
069     * <ul>
070     * <li>
071     * Use the <code>javax.xml.parsers.DocumentBuilderFactory</code> system
072     * property.
073     * </li>
074     * <li>
075     * Use the properties file "lib/jaxp.properties" in the JRE directory.
076     * This configuration file is in standard <code>java.util.Properties
077     * </code> format and contains the fully qualified name of the
078     * implementation class with the key being the system property defined
079     * above.
080     *
081     * The jaxp.properties file is read only once by the JAXP implementation
082     * and it's values are then cached for future use.  If the file does not exist
083     * when the first attempt is made to read from it, no further attempts are
084     * made to check for its existence.  It is not possible to change the value
085     * of any property in jaxp.properties after it has been read for the first time.
086     * </li>
087     * <li>
088     * Use the Services API (as detailed in the JAR specification), if
089     * available, to determine the classname. The Services API will look
090     * for a classname in the file
091     * <code>META-INF/services/javax.xml.parsers.DocumentBuilderFactory</code>
092     * in jars available to the runtime.
093     * </li>
094     * <li>
095     * Platform default <code>DocumentBuilderFactory</code> instance.
096     * </li>
097     * </ul>
098     *
099     * Once an application has obtained a reference to a
100     * <code>DocumentBuilderFactory</code> it can use the factory to
101     * configure and obtain parser instances.
102     *
103     *
104     * <h2>Tip for Trouble-shooting</h2>
105     * <p>Setting the <code>jaxp.debug</code> system property will cause
106     * this method to print a lot of debug messages
107     * to <code>System.err</code> about what it is doing and where it is looking at.</p>
108     *
109     * <p> If you have problems loading {@link DocumentBuilder}s, try:</p>
110     * <pre>
111     * java -Djaxp.debug=1 YourProgram ....
112     * </pre>
113     *
114     * @return New instance of a <code>DocumentBuilderFactory</code>
115     *
116     * @throws FactoryConfigurationError if the implementation is not
117     *   available or cannot be instantiated.
118     */
119    public static DocumentBuilderFactory newInstance() {
120        try {
121            return (DocumentBuilderFactory) FactoryFinder.find(
122                /* The default property name according to the JAXP spec */
123                "javax.xml.parsers.DocumentBuilderFactory",
124                /* The fallback implementation class name */
125                "com.sun.org.apache.xerces.internal.jaxp.DocumentBuilderFactoryImpl");
126        } catch (FactoryFinder.ConfigurationError e) {
127            throw new FactoryConfigurationError(e.getException(),
128                                                e.getMessage());
129        }
130
131    }

147     *
148     * <p> If you have problems try:</p>
149     * <pre>
150     * java -Djaxp.debug=1 YourProgram ....
151     * </pre>
152     *
153     * @param factoryClassName fully qualified factory class name that provides implementation of <code>javax.xml.parsers.DocumentBuilderFactory</code>.
154     *
155     * @param classLoader <code>ClassLoader</code> used to load the factory class. If <code>null</code>
156     *                     current <code>Thread</code>'s context classLoader is used to load the factory class.
157     *
158     * @return New instance of a <code>DocumentBuilderFactory</code>
159     *
160     * @throws FactoryConfigurationError if <code>factoryClassName</code> is <code>null</code>, or

163     * @see #newInstance()
164     *
165     * @since 1.6
166     */
167    public static DocumentBuilderFactory newInstance(String factoryClassName, ClassLoader classLoader){
168        try {
169            //do not fallback if given classloader can't find the class, throw exception
170            return (DocumentBuilderFactory) FactoryFinder.newInstance(factoryClassName, classLoader, false);
171        } catch (FactoryFinder.ConfigurationError e) {
172            throw new FactoryConfigurationError(e.getException(),
173                                                e.getMessage());
174        }

419        /**
420         * <p>Set a feature for this <code>DocumentBuilderFactory</code> and <code>DocumentBuilder</code>s created by this factory.</p>
421         *
422         * <p>
423         * Feature names are fully qualified {@link java.net.URI}s.
424         * Implementations may define their own features.
425         * A {@link ParserConfigurationException} is thrown if this <code>DocumentBuilderFactory</code> or the
426         * <code>DocumentBuilder</code>s it creates cannot support the feature.
427         * It is possible for a <code>DocumentBuilderFactory</code> to expose a feature value but be unable to change its state.
428         * </p>
429         *
430         * <p>
431         * All implementations are required to support the {@link javax.xml.XMLConstants#FEATURE_SECURE_PROCESSING} feature.
432         * When the feature is:</p>
433         * <ul>
434         *   <li>
435         *     <code>true</code>: the implementation will limit XML processing to conform to implementation limits.
436         *     Examples include enity expansion limits and XML Schema constructs that would consume large amounts of resources.
437         *     If XML processing is limited for security reasons, it will be reported via a call to the registered
438         *    {@link org.xml.sax.ErrorHandler#fatalError(SAXParseException exception)}.
439         *     See {@link  DocumentBuilder#setErrorHandler(org.xml.sax.ErrorHandler errorHandler)}.
440         *   </li>
441         *   <li>
442         *     <code>false</code>: the implementation will processing XML according to the XML specifications without
443         *     regard to possible implementation limits.
444         *   </li>
445         * </ul>
446         *
447         * @param name Feature name.
448         * @param value Is feature state <code>true</code> or <code>false</code>.
449         *
450         * @throws ParserConfigurationException if this <code>DocumentBuilderFactory</code> or the <code>DocumentBuilder</code>s
451         *   it creates cannot support this feature.
452     * @throws NullPointerException If the <code>name</code> parameter is null.
453         */
454        public abstract void setFeature(String name, boolean value)
455                throws ParserConfigurationException;

527    /**
528     * <p>Set the {@link Schema} to be used by parsers created
529     * from this factory.
530     *
531     * <p>
532     * When a {@link Schema} is non-null, a parser will use a validator
533     * created from it to validate documents before it passes information
534     * down to the application.
535     *
536     * <p>When errors are found by the validator, the parser is responsible
537     * to report them to the user-specified {@link org.xml.sax.ErrorHandler}
538     * (or if the error handler is not set, ignore them or throw them), just
539     * like any other errors found by the parser itself.
540     * In other words, if the user-specified {@link org.xml.sax.ErrorHandler}
541     * is set, it must receive those errors, and if not, they must be
542     * treated according to the implementation specific
543     * default error handling rules.
544     *
545     * <p>
546     * A validator may modify the outcome of a parse (for example by
547     * adding default values that were missing in documents), and a parser
548     * is responsible to make sure that the application will receive
549     * modified DOM trees.
550     *
551     * <p>
552     * Initialy, null is set as the {@link Schema}.
553     *
554     * <p>
555     * This processing will take effect even if
556     * the {@link #isValidating()} method returns <code>false</code>.
557     *
558     * <p>It is an error to use
559     * the <code>http://java.sun.com/xml/jaxp/properties/schemaSource</code>
560     * property and/or the <code>http://java.sun.com/xml/jaxp/properties/schemaLanguage</code>
561     * property in conjunction with a {@link Schema} object.
562     * Such configuration will cause a {@link ParserConfigurationException}
563     * exception when the {@link #newDocumentBuilder()} is invoked.</p>
564     *
565     *
566     * <h4>Note for implmentors</h4>
567     *
568     * <p>
569     * A parser must be able to work with any {@link Schema}
570     * implementation. However, parsers and schemas are allowed
571     * to use implementation-specific custom mechanisms
572     * as long as they yield the result described in the specification.
573     * </p>
574     *
575     * @param schema <code>Schema</code> to use or <code>null</code>
576     *   to remove a schema.
577     *
578     * @throws UnsupportedOperationException When implementation does not
579     *   override this method.
580     *
581     * @since 1.5
582     */
583    public void setSchema(Schema schema) {
584        throw new UnsupportedOperationException(
585            "This parser does not support specification \""
586            + this.getClass().getPackage().getSpecificationTitle()
587            + "\" version \""
588            + this.getClass().getPackage().getSpecificationVersion()
589            + "\""
590            );
591    }
