055/**
056 * <p>RMI object used to forward an MBeanServer request from a client
057 * to its MBeanServer implementation on the server side.  There is one
058 * Remote object implementing this interface for each remote client
059 * connected to an RMI connector.</p>
060 *
061 * <p>User code does not usually refer to this interface.  It is
062 * specified as part of the public API so that different
063 * implementations of that API will interoperate.</p>
064 *
065 * <p>To ensure that client parameters will be deserialized at the
066 * server side with the correct classloader, client parameters such as
067 * parameters used to invoke a method are wrapped in a {@link
068 * MarshalledObject}.  An implementation of this interface must first
069 * get the appropriate class loader for the operation and its target,
070 * then deserialize the marshalled parameters with this classloader.
071 * Except as noted, a parameter that is a
072 * <code>MarshalledObject</code> or <code>MarshalledObject[]</code>
073 * must not be null; the behavior is unspecified if it is.</p>
074 *
075 * <p>Class loading aspects are detailed in the
076 * <a href="{@docRoot}/../technotes/guides/jmx/JMX_1_4_specification.pdf">
077 * JMX Specification, version 1.4</a> PDF document.</p>
078 *
079 * <p>Most methods in this interface parallel methods in the {@link
080 * MBeanServerConnection} interface.  Where an aspect of the behavior
081 * of a method is not specified here, it is the same as in the
082 * corresponding <code>MBeanServerConnection</code> method.
083 *
084 * @since 1.5
085 */
086/*
087 * Notice that we omit the type parameter from MarshalledObject everywhere,
088 * even though it would add useful information to the documentation.  The
089 * reason is that it was only added in Mustang (Java SE 6), whereas versions
090 * 1.4 and 2.0 of the JMX API must be implementable on Tiger per our
091 * commitments for JSR 255.  This is also why we suppress rawtypes warnings.
092 */
